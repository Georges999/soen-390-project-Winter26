# Sprint Documentation

This folder contains sprint planning, reports, and retrospectives.

## Sprint Structure

Each sprint should have a folder with the following documents:

```
sprint-1/
├── planning.md         # Sprint goals and task breakdown
├── daily-standups.md   # Daily standup notes
├── report.md           # Sprint achievements and metrics
└── retrospective.md    # What went well, what to improve
```

## Sprint Template

### Planning Document
- Sprint goals
- User stories
- Task breakdown
- Team assignments
- Definition of done

### Daily Standups
- Date
- Attendees
- What was accomplished
- What's planned for today
- Blockers

### Sprint Report
- Goals achieved
- Metrics (velocity, burndown)
- Completed features
- Technical debt addressed
- Challenges faced

### Retrospective
- What went well
- What could be improved
- Action items for next sprint
- Team morale
